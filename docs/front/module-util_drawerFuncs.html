(static) drawNumPostsOverTime(data, setCurrGraphTime)

Draws line chart for number of Tweets the selected/clicked user has postde over the default or user-specified date range. Code skeleton from here. Creating labels for the pie segments came from here.

Parameters:
Name Type Description
data Object
Properties
Name Type Description
colors Array

Each element looks like '#7878FF'. Each color depends on the average overall Tweet polarity scores within a time step for a certain user. Blue corresponds to very liberal-leaning posts and red corresponds to very conservative-leaning posts.
range: Array

A 2-element array that consists of [Least number of tweets posted within a day by a certain user, Greatest number of tweets posted within a day by a certain user].
sizes Array

Radii of each point on the line chart, each corresponding to how many posts a user tweets within a single time step.
stops Array

Real numbers from 0 to 100 that mark what percentages of the x-axis the color of the line chart should change at. E.g., [0, 11.2, 45.6, 100]
times Array

Datetime strings of the form 'MM/DD/YYYY, hh:mm:ss' that each correspond to a point on the line chart.
setCurrGraphTime function

When user clicks on a point in the line chart, this function moves the Timeline to the time instance corresponding to that point on the line chart.
